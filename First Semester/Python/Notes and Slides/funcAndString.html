Functions? Why use functions?¶

  • Functions are a group of statements, so they can be run more than once in a program.
  • Just like a value can be associated with a name, a piece of logic can also be associated with a name by defining a function.
  • Functions serve two primary development roles:

    Code reuse¶

    • Functions allow us to group and parametrize chunks of code to be used arbitrarily many times later.

    Procedural decomposition/Modularity¶

    • Functions also provide a tool for splitting systems into pieces that have a well-defined role.

    Variable Scope¶

    • Global variables are variables declared outside functions and can be accessed from anywhere within the program
    • Local variables are variables declared inside a function and can be used only inside that particular function
    • from inside a function, a variable which is defined outside the function can be accessed even if not explicitly passed as arguments
    • locals variables cannot be accessed from outside functions
  • When Python sees use of a variable not defined locally, it tries to find a global variable with that name.
  • However, you have to explicitly declare a variable as global to modify it.

    Solving problems with Recursion¶

    • One way to describe repetition within a computer program is the use of loops, such as python's while-loop and for-loop.
    • An entirely different way to achieve repetition is through a process called *recursion*.
    • Recursion is a technique by which a function makes one or more calls to itself during execution.
    • In computing, recursion provides an elegant and powerful alternative for performing repetitive tasks.
    • When one invocation of the function makes a recursive call, that invocation is suspended until the recursive call completes.
    In [28]:
    # iterative appoarch
def factorial_iterative(n):
    fact = 1
    for i in range(n,0,-1):
        fact*=i
    return fact

factorial_iterative(5)
    Out[28]:
    120

    A natural recursive definition for the factorial function.¶

    $$ 5! = 5.(4.3.2.1) = 5.4! $$

    More generally, for a positive integer $n$, we can define $n!$ to be $n.(n-1)!$

    \begin{equation*} n! = \begin{cases} 1 &\text{if n = 0 }\\ n.(n-1)! &\text{if n>= 1} \end{cases} \end{equation*}
    • The definition is typical of many recursive definitions.
    • First it contains one or more *base cases*, in this case $n=0$ is the base case.
    • It also contains one or more *recursive cases*, which are defined by appealing to the definition of the function being defined.
    In [29]:
    # recursive implementation
def factorial_recursive(n):
    if n == 0:
        return 1
    else:
        return n*factorial_recursive(n-1)

factorial_recursive(5)
    Out[29]:
    120

    Recursive algorithm for computing powers¶

    • Compute the power function, defined as power(x,n) = x^n
    • Observe the fact that $x^n = x.x^{n-1}$ for $n>0$
    \begin{equation*} power(x,n) = \begin{cases} 1 &\text{if n = 0 }\\ x.power(x,n-1) &\text{otherwise} \end{cases} \end{equation*}
    In [30]:
    def power(x,n):
    if n == 0:
        return 1
    else:
        return x * power(x,n-1)
    
power(2,4)
    Out[30]:
    16

    Python Lambda/Anonymous Function¶

    In Python, a lambda function is a special type of function without the function name.¶

    A lambda function can take any number of arguments, but can only have one expression.¶

    Syntax¶

    lambda arguments : expression¶

    In [40]:
    greet=lambda : print('Hello World')
greet()

    Hello World
    In [41]:
    x = lambda a : a + 10
print(x(5))

    15
    Write a lambda function to calculate cube of a number¶
    In [42]:
    cube = lambda a : a * a * a
print(cube(5))

    125
    In [43]:
    greet1=lambda name: print("hello", name)
greet1('ram')

    hello ram

    Lambda functions can take any number of arguments:¶

    In [44]:
